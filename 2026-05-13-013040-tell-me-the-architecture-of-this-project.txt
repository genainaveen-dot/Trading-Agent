● Project Architecture: Demand/Supply Trading Agent

  This is a Python trading system with these core layers:

  Core Components

  1. Strategy Engine (trading_agent/strategy/)
  - candles.py - Candle classification
  - zones.py - Demand/supply zone detection
  - trend.py - Trend analysis
  - curve.py - Curve analysis
  - scoring.py - Signal scoring
  - engine.py - Strategy orchestration

  2. Risk Management (trading_agent/risk.py)
  - Position sizing based on account equity
  - Daily loss limits (4.5%)
  - Max open trades limit (3)
  - Trade rejection logic

  3. Execution Layer (trading_agent/execution/)
  - base.py - Abstract ExecutionAdapter interface
  - paper.py - Paper trading (safe mode)
  - kite.py - Zerodha Kite adapter (legacy)
  - dhan.py - DhanHQ live broker adapter

  4. Supporting Modules
  - market_data.py - Market data fetching
  - backtest.py - Backtesting engine
  - alerts.py - Swing alert generation
  - utils.py - Utilities

  5. API/Dashboard (trading_agent/api/)
  - REST endpoints for signals, alerts, positions, orders, risk
  - Visual dashboard (dashboard tab)
  - Kill switch endpoint

  Configuration

  - config.yaml - Main config (broker, risk params)
  - .env - Credentials (DHAN_CLIENT_ID, DHAN_ACCESS_TOKEN)
  - instruments.dhan.csv - Symbol to security_id mapping

  Safe Defaults

  Default mode is paper trading with live_enabled: false. Live orders require:
  correct mode, valid credentials, instrument mappings, and passing risk checks.

● Trading Flow

  ┌─────────────────────────────────────────────────────────────────────────┐

  ┌─────────────────────────────────────────────────────────────────────────┐
  │                          TradingAgent (agent.py)                        │
  │                                                                          │
  │  1. process_symbol() receives candles for a symbol                      │
  └─────────────────────────────────┬───────────────────────────────────────┘
                                    │
                                    ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                     StrategyEngine (strategy/engine.py)                 │
  │                                                                          │
  │  2. analyze_trend()     → TrendState (UP/DOWN/SIDEWAYS)                 │
  │  3. detect_zones()      → Demand/Supply zones from candles              │
  │  4. score_zone()        → Assigns score based on pattern strength       │
  │  5. entry_type_for_score() → Reject low-score signals                   │
  │  6. Curve analysis      → HIGH/LOW/EQUILIBRIUM relative to HTF zones    │
  │                                                                          │
  │  Output: TradeSignal with entry, stop_loss, target, score               │
  └─────────────────────────────────┬───────────────────────────────────────┘
                                    │
                                    ▼
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                      RiskManager (risk.py)                              │
  │                                                                          │
  │  Checks:                                                                 │
  │  • Kill switch enabled?                                                  │
  │  • Live mode disabled?                                                   │
  │  • Max open trades reached? (default: 3)                                 │
  │  • Daily loss exceeded? (default: 4.5%)                                 │

  │  • Signal score < 5?                                                    │

  │                                                                          │

  Key Decision Points

  1. Signal Generation: Only high-scoring demand/supply zones with favorable
  trend/curve conditions become signals
  2. Risk Gate: Every signal must pass multiple checks before an order is placed
  3. Execution Mode:
    - Paper: Simulates fills immediately
    - Live: Routes to DhanHQ with real orders
  4. Swing Alert Agent: For swing timeframe on Dhan broker, valid signals also
  generate alerts (non-execution notifications)
